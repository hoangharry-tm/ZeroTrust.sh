# decisions/

This folder tracks the current state of assumptions and risks — what the analysis is betting on, and what could go wrong. `assumptions.md` lists every assumption baked into the proposals with a validation status and plan. `risk_registry.md` tracks risks across Technical, Market, Legal, and Operational categories. Neither file records decisions that have been made — those go into `specs/architecture/adr/`. This folder answers the question: "what do we still need to prove before we can confidently proceed?"
