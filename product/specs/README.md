# specs/

This folder contains all product and technical specifications for ZeroTrust.sh. It is divided into three subfolders: `proposals/` for competing architectural approach documents (the core analysis deliverable), `architecture/` for structural reference documents and Architecture Decision Records, and `features/` for individual feature specifications that will be written after approach selection. Nothing in `specs/architecture/adr/` should be written until the tech lead selects an approach — ADRs record decisions already made, not options under consideration.
